## How It Works

```
Step 1: score = (weight₁ × feature₁) + (weight₂ × feature₂) + ... + bias

Step 2: probability = sigmoid(score) = 1 / (1 + e^(-score))
```

Large positive score → probability near 1 (will happen)  
Large negative score → probability near 0 (won't happen)  
Score near 0 → probability near 0.5 (coin flip)

## Reading Coefficients

<table fit-page-width="true" header-row="true">
<tr><td>Coefficient</td><td>Meaning</td></tr>
<tr><td>**Positive** (+)</td><td>Feature increases probability of outcome</td></tr>
<tr><td>**Negative** (-)</td><td>Feature decreases probability of outcome</td></tr>
<tr><td>**Larger magnitude**</td><td>Stronger effect</td></tr>
<tr><td>**Near zero**</td><td>Weak or no effect</td></tr>
</table>

**Example:**
```
support_tickets:  +0.72  → More tickets = higher churn risk
tenure_months:    -0.25  → Longer tenure = lower churn risk
```

**Warning:** Compare magnitudes only if features are standardized (same scale).

## When to Use Logistic Regression

<table fit-page-width="true" header-row="true">
<tr><td>✅ Good fit</td><td>❌ Not ideal</td></tr>
<tr><td>Need interpretable coefficients</td><td>Complex feature interactions</td></tr>
<tr><td>Limited data (<10K rows)</td><td>Non-linear patterns</td></tr>
<tr><td>Want a simple baseline</td><td>Tons of data (1M+ rows)</td></tr>
<tr><td>Relationships are roughly linear</td><td>Need absolute best accuracy</td></tr>
</table>

**Default approach:** Start with logistic regression. Upgrade only if it's not good enough.

## Quick Code Reference

```python
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import train_test_split

# Split data
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Train
model = LogisticRegression()
model.fit(X_train, y_train)

# Predict probabilities
probs = model.predict_proba(X_test)[:, 1]

# Get coefficients
for feature, coef in zip(features, model.coef_[0]):
    print(f"{feature}: {coef:+.3f}")
```

## Common Mistakes

<table fit-page-width="true" header-row="true">
<tr><td>Mistake</td><td>Symptom</td><td>Fix</td></tr>
<tr><td>**Multicollinearity**</td><td>Coefficients flip signs unexpectedly</td><td>Check correlations, remove redundant features</td></tr>
<tr><td>**Unscaled features**</td><td>Can't compare coefficient magnitudes</td><td>Use StandardScaler</td></tr>
<tr><td>**Treating 51% like certainty**</td><td>Acting on borderline predictions</td><td>Use ranking, not thresholds</td></tr>
<tr><td>**Expecting magic**</td><td>"Only 0.71 AUC, need better model"</td><td>Check baselines first—0.71 might be good</td></tr>
</table>

## Precision@K Formula

When you can only act on K customers:

```python
# Sort by probability (highest first)
top_k_indices = np.argsort(probabilities)[-k:]

# What fraction actually had the outcome?
precision_at_k = y_test.iloc[top_k_indices].mean()

# Compare to baseline
lift = precision_at_k / y_test.mean()
```

## Linear vs. Logistic

<table fit-page-width="true" header-row="true">
<tr><td></td><td>Linear Regression</td><td>Logistic Regression</td></tr>
<tr><td>**Predicts**</td><td>Continuous number</td><td>Probability [0, 1]</td></tr>
<tr><td>**Use for**</td><td>LTV, revenue, time</td><td>Churn yes/no, fraud yes/no</td></tr>
<tr><td>**Output**</td><td>weighted sum</td><td>sigmoid(weighted sum)</td></tr>
</table>

## Coefficient Sanity Check

Before trusting coefficients, verify:

- [ ] Feature scales are similar (or standardized)
- [ ] No high correlations between features (< 0.8)
- [ ] Signs match intuition (tenure should be protective)
- [ ] No data leakage in features

If coefficients don't make sense, the model might be right but your intuition wrong—or there's a data problem.
